Week 8 - Fetch jQuery

  • In Week 8, I learnt that jQuery doesn't have a specific "Fetch" function, its robust AJAX methods like 'span class="math-inline"\.ajax\(\)\' and convenient shortcuts like `.get()` empower you to retrieve data from servers asynchronously. These methods handle various request types, which allows us to send data to the server and receive a response without refreshing the page. It lets us build dynamic web experiences where user interaction triggers data exchanges behind the scenes, enriching our applications and keeping users engaging. jQuery's AJAX methods come with benefits like cross-browser compatibility and seamless integration with other features like DOM manipulation and event handling, making them flexible and powerful tools for fetching data in our web projects.
  • I also learnt the terms Data Attribute and API during this week. Data attributes are used to store custom data private to the page or application. They provide a way to embed custom data private to the page or application that can be used by JavaScript or CSS. Data attributes are prefixed with "data-" and can be added to any HTML element.
  • APIs in HTML usually refer to the JavaScript APIs that browsers provide to interact with different aspects of a web page or to communicate with external services. Common APIs include the DOM (Document Object Model) API, which allows manipulation of HTML and XML documents, and the Fetch API, which is used for making HTTP requests.
  • All in all, data attributes allow us to embed custom data in HTML elements, while APIs enable communication and interaction with external services. Combining data attributes with APIs can help create dynamic and personalised web applications.

Week 12 - RESTDB & NoSQL

  • In Week 12, i was taught about the concepts of RESTful databases (RestDB) and NoSQL have emerged as pivotal elements, reshaping the way data is managed and accessed in HTML applications. This reflection aims to unravel the significance of RestDB and NoSQL within the context of HTML, shedding light on their roles in modern web development.
  • RestDB, short for RESTful Databases, epitomizes a paradigm shift in how web applications communicate with databases. The term "RESTful" stands for Representational State Transfer, a set of principles that guide the design of networked applications. In the context of RestDB, this entails a standardized approach to creating, reading, updating, and deleting data via HTTP methods, mirroring the CRUD operations. The essence of RestDB lies in its simplicity and adherence to REST principles, fostering a stateless communication model. As I incorporate RestDB into my HTML projects, the clean and intuitive API endpoints streamline data interactions, promoting a seamless connection between the front-end and the database. This aligns with the ethos of REST, making my applications more scalable, maintainable, and interoperable. As I integrate NoSQL databases into my HTML projects, I appreciate the agility they bring to the development process. The absence of rigid schemas empowers me to adapt to changing requirements without the constraints of predefined table structures. This flexibility proves invaluable when dealing with diverse data formats within HTML applications, allowing for efficient storage and retrieval of information.

Week 14 - Lottie

  • In Week 13, I learnt that Lottie, developed by Airbnb, is an open-source animation library that allows developers to seamlessly incorporate high-quality animations into web and mobile applications. At its core, Lottie is based on JSON (JavaScript Object Notation) files that encapsulate vector-based animations created in design tools like Adobe After Effects. This file format ensures a lightweight and scalable approach to animations, paving the way for a more engaging and interactive user experience.
  • Also, i learnt that as I embrace Lottie within my HTML projects, the integration process proves to be remarkably straightforward and efficient. The library provides a bridge between designers and developers by enabling the use of animations created by designers without the need for extensive coding efforts. The JSON-based animations are easily embedded into HTML, and the Lottie library takes care of rendering them seamlessly across various devices and browsers. One of the notable features of Lottie in HTML is its platform-agnostic nature. The animations play consistently across different devices, offering a cohesive and uniform user experience. This aspect is particularly crucial in the diverse landscape of web development, where ensuring a consistent visual appeal is paramount.
